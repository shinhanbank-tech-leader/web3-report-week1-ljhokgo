# 제목: 블록체인의 신뢰 형성 구조와 기존 금융 시스템의 신뢰 구조 비교

## 부서명: 디지털서비스개발부
## 이름: 이재훈

-------------------

## 레포트 내용

국내 기존 금융 시스템은 한국은행·금융위원회·금융감독원 등 중앙 기관의 권위와 법적 보증에 기반한 **위임된 신뢰(Delegated Trust)** 구조를 갖는다. 반면 블록체인은 수학적 암호화와 분산 합의 알고리즘을 통해 제3자 기관 없이 신뢰를 형성하는 **탈중앙화 신뢰(Decentralized Trust)** 패러다임을 구현한다. 두 구조는 상호 배타적이지 않으며, 국내 금융 환경에서는 하이브리드 결합이 가장 현실적인 진화 방향이다.

### 신뢰 구조 핵심 비교

| 비교 항목 | 기존 금융 시스템 | 블록체인 기반 |
| --- | --- | --- |
| 신뢰의 원천 | 중앙 기관의 권위·법적 보증 | 수학적 암호화·합의 알고리즘 |
| 검증 주체 | 한국은행·금감원·시중은행 | 분산 네트워크 노드 전체 |
| 원장 구조 | 중앙화 원장 (기관 개별 보유) | 분산원장 (전 참여자 공유) |
| 투명성 | 제한적 (기관 내부) | 완전 공개 또는 허가형 공개 |
| 결제 최종성 | T+2일(주식), 당일(RTGS) | 즉시 (합의 완료 후) |
| 해외 송금 | 1~5일 · 수수료 약 6.3% | 수 초 · 수수료 0.01% 미만 |
| 장애 복원력 | 단일실패점(SPoF) 존재 | 고가용성 (노드 분산) |
| 소비자 보호 | 예금보험·분쟁조정 완비 | 코드 의존, 보호 체계 미성숙 |
| 규제 적합성 | 완전 적합 | 부분적 (샌드박스 적용 중) |
| 처리 속도 | 수천~수만 TPS | 7~15 TPS(퍼블릭) / 수천(프라이빗) |

### 기존 금융 시스템 — 강점과 한계

**강점**

- **법적 확실성**: 모든 거래에 법적 효력 부여, 분쟁 해결 체계 명확
- **소비자 보호**: 예금자보호(1인당 최고 5,000만 원), 금융소비자보호법 등 다층 보호
- **운영 안정성**: BOK-Wire+(일평균 270조 원 처리) 등 수십 년 검증된 인프라

**한계**

- **중앙화 리스크**: 핵심 인프라 사이버 공격·장애 시 대규모 피해 가능
- **비용 구조**: 중개기관 유지·결제 수수료·규제 컴플라이언스 비용 지속 발생
- **국경 간 비효율**: SWIFT 경유 해외 송금은 수 일 소요 및 높은 수수료 부과

### 블록체인 — 강점과 한계

**강점**

- **탈중앙화**: 단일실패점 없이 수천 개 노드가 동시 검증, 조작 사실상 불가
- **효율성**: 스마트 컨트랙트로 무역금융·DVP 결제 자동화, T+0 결제 구현 가능
- **비용 절감**: 중간 기관 제거로 해외 송금 비용·결제 인프라 비용 획기적 감소

**한계**

- **확장성**: 퍼블릭 체인의 낮은 TPS(7~15건)로 대량 금융 거래 처리 한계
- **스마트 컨트랙트 취약점**: 코드 버그 시 되돌릴 수 없는 금전 손실 위험
- **규제 불확실성**: 스마트 컨트랙트 법적 효력, 잊혀질 권리 충돌 등 미해결


### 국내 적용 현황 및 결론

국내에서는 ① 한국은행 CBDC 파일럿(제도적 신뢰 + 블록체인 효율 결합), ② DID 기반 금융 인증(뱅크사인), ③ 무역금융 블록체인(서류 검토 수 일 → 수 시간), ④ 금융결제원 블록체인 오픈플랫폼(BOP) 등 하이브리드 모델이 확산 중이다. 특금법·가상자산이용자보호법(2024) 시행으로 제도적 기반도 정비되고 있다.

블록체인은 기존 금융 시스템을 대체하는 것이 아니라 보완하는 역할로 접근할 때 가장 큰 가치를 창출한다. 규제 선명성 확보, 공공 인프라 표준화, 글로벌 정합성 확보를 병행할 때 두 신뢰 구조의 융합이 국내 금융 시스템의 다음 단계를 열 수 있다 생각한다.
